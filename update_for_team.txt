WHAT WE'RE DOING (quick recap)
--------------------------------
Quick review, the Silent Branding Attack where someone poisons a training dataset by sneaking a logo into a bunch of images really subtly. If you then finetune a diffusion model on those images, the model picks up the logo and starts putting it in basically every image it generates. Note, no trigger, no special prompt, it just always shows up. The use case: some random, unknowing user would train on what looks like a normal dataset and suddenly every image their model makes has the Avengers logo on someone's shirt or bag.

The research goal is to detect this: Can we tell if a model got poisoned just from looking at its outputs?

---

WHAT WE FOUND
--------------
Short answer: yes and it works really well. 
Concern: could a real defender use it?

Here's the idea. We generate 1000 random images from a suspect model and look at them in "frequency space" (basically analyzing the fingerprint of the pixel patterns rather than what the images look like). We find the logo leaves a consistent signature, but in any single image you can't see really define it, it's buried under all the other content. BUT, when we average the frequency patterns across all 1000 images, the random content cancels out and the logo's fingerprint is what's left.

Then we train and then give that fingerprint to a small neural net and ask: poisoned or clean? It works perfectly for this case.

---

THE RESULTS
------------
- AUROC = 1.0. So a perfect score. Not a single error on the test set, every time. I ran a validation script (testing every possible "cheat" or "hack" the code could have that     ruins results but it passed everything so good for now)
- Works with as few as 100 images actually. Don't need 10,000 for sure. We will keep just 1000 to have it nice and safe.
- Tried three different ways to analyze frequencies (DCT, FFT, wavelets). All almost perfect. This is good and really proves it's VERY detectable.
- Tried different ways to average (mean, median, trimmed mean). All AUROC = 1.0. Doesn't matter which way we even try to define the fingerprint.

---

THE WILD MODEL TEST (this is the important experiment)
------------------------------------------
We need to make sure we're not just flagging every finetuned model as poisoned. So we use Juggernaut-XL (a random model with like a million downloads and apparently a really good photorealistic model someone trained) to test this.

First try: the classifier said 99.7% poisoned. Oops.

Problem was the classifier only knew one type of "clean" model, so anything that looked different got flagged. The fix was to retrain it with Juggernaut in the clean pool so it learns that clean means any legit model, not just the one we happened to test against.

After the fix: Juggernaut false alarm rate = 0%. Poisoned model detection = 100%. AUROC still = 1.0.

Honestly this ended up being a good result for the paper. Shows the method actually learned what a poisoned fingerprint looks like rather than just memorizing what our one clean model looks like.

WHAT'S STILL RUNNING

If it works, the method generalizes across logos too. This is a VERY important result if it works in our favor.

---

BOTTOM LINE FOR THE PAPER
---------------------------
We have:
1. A detector that works perfectly (AUROC = 1.0)
2. It's robust across frequency methods and aggregation strategies — same result no matter what
3. It generalizes to unseen clean model types (the Juggernaut fix)
4. A clean story: averaging across a big population cancels out content and surfaces the logo fingerprint

Every existing defense out there relies on triggers. This attack has no trigger, so they all fail. We're the first to show a working detector for this specific scenario. That's the paper.
